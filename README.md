## Engineering Philosophy

> 화면에서 시작된 요청이 API, DB, 권한, 상태 변경까지 어긋나지 않게 이어지는 구조를 지향합니다.

- 기능을 만들기 전에 사용자 흐름, 필요한 데이터, 예외 상황을 먼저 정리합니다.
- 과도한 추상화보다 읽기 쉬운 계층 분리와 명확한 응답 구조를 선호합니다.
- 프론트엔드와 백엔드가 함께 움직일 때 API 계약, 상태값, 권한 흐름을 꼼꼼히 확인합니다.
- 모르는 부분은 질문하고 기록하며, 팀원이 다시 확인하기 쉬운 문서를 남기려 합니다.
